# API Specification
All routes below are prefixed `/api` and (except auth register/login) require Sanctum bearer auth.

- `/auth` ‚Äî register, login, me, logout, fcm-token, `PUT /auth/password` (change password: `current_password`, `new_password`, `new_password_confirmation` ‚Äî requires the current password, revokes every other active token on success). Also public (throttled 5/10min): `POST /auth/forgot-password` (`email` ‚Üí generic message, sends a 6-digit OTP via WhatsApp if the account exists and has a phone on file) and `POST /auth/reset-password` (`email`, `otp`, `new_password`, `new_password_confirmation` ‚Üí revokes every active token on success) ‚Äî see AGENTS.md "Forgot-password via WhatsApp OTP".
- `/contacts` ‚Äî CRUD (index, store, show, update, destroy). `index` defaults to alphabetical; pass `?for_chat=1` for a "recent chats" ordering instead ‚Äî only contacts with at least one message, most recently messaged first, with `latest_message` eager-loaded on each (`{id, direction, content, created_at}`) for a preview. Powers `jarwo_app`'s "Chat" tab.
- `/contacts/{contact}/messages` ‚Äî index, store (send via WhatsApp gateway), destroy (DELETE, body `{"ids": [1,2,3]}` ‚Äî bulk-deletes the app's own record of those messages; does not un-send/recall them on WhatsApp itself, there's no API for that)
- `/contacts/{contact}/messages/import` ‚Äî upload a WhatsApp "Export Chat" .txt file (multipart: `file`, `owner_name`) to backfill historical messages for a contact. Response (`201`) returns immediately with `imported_count`/`skipped_count` and `"analysis": "queued"` ‚Äî memory extraction + style profile analysis run in a background job (`App\Jobs\AnalyzeContactHistory`), not inline, so a large import's AI calls can't make the request itself time out. This is how the AI learns the *owner's* actual voice ‚Äî normal webhook traffic never captures what the owner types from their own phone, only inbound messages.
- `/contacts/{contact}/suggest-reply` ‚Äî AI-generated reply suggestion
- `/contacts/{contact}/memories/analyze` ‚Äî extract memories + refresh style profile from recent conversation
- `/contacts/{contact}/style-profile` (GET) ‚Äî read the contact's current `StyleProfile` row (`{"style_profile": null}` if no analysis has ever run). No AI call ‚Äî a passive read counterpart to `memories/analyze`, added for the `jarwo_app` mobile client so viewing a contact's style doesn't force a costly re-analysis.
- `/memories` ‚Äî index (filter by contact_id/type), store, destroy
- `/events` ‚Äî CRUD, local schedule/reminder entries (`title`, `description`, `start_at`, `end_at`, optional `contact_id`). `index`/`show`/`store`/`update` all eager-load `contact:id,name` so a client can show which contact an event relates to without a second request. `index` defaults to `upcoming=1` (only future events, ordered soonest-first); pass `upcoming=0` for the full history. Distinct from Google Calendar sync (not implemented ‚Äî no OAuth/Calendar API code exists yet), though the model has `google_event_id`/`source` columns reserved for that later.
- `/settings` ‚Äî index, update (upsert by key), destroy. Notable keys: `auto_reply_enabled` (global auto-reply switch), `default_persona` (free-text fallback style used for contacts with no `StyleProfile` yet ‚Äî see AGENTS.md "Default persona + periodic auto-reanalysis"), `sensitive_topics_guard` (free-text topics unattended auto-reply must withhold on, e.g. money/approval requests ‚Äî see AGENTS.md "Sensitive-topic withhold guard").
- `/webhooks/fonnte/{secret}` ‚Äî **public** (no Sanctum), verified by a shared secret path segment instead. Receives inbound WhatsApp messages from Fonnte, creates/updates the Contact, stores the Message, fires an AI auto-reply only if both the owner's global `auto_reply_enabled` setting AND the contact's `ai_enabled` flag are on, and independently dispatches `App\Jobs\AnalyzeContactHistory` once enough new messages have accumulated for that contact (see AGENTS.md).
- `/contacts/{contact}` (update) also accepts `ai_enabled` (boolean) to toggle auto-reply for that one contact, independent of the global setting.

Not yet implemented: Google Calendar sync endpoints, Wablas inbound webhook (Fonnte only so far).
